# Проверочный лист безопасности бизнес-систем PropDevelopment

## Обоснование выбора разделов

Исходя из проблем компании, приоритетные разделы:

1. **Управление доступом** — несколько точек регистрации клиентов без координации, партнёры видят чужие данные
2. **Безопасность данных** — нарушен контроль данных, отображение чужих ФИО, ПДн без единых политик
3. **Защита инфраструктуры** — нет единых политик для API партнёров, гибридная инфраструктура
4. **Управление инцидентами** — единственный ИБ-специалист не получает информацию о всех изменениях

## Выводы

**Критические проблемы:**
1. Отсутствует единый реестр пользователей и централизованное управление доступом
2. Нет контроля целостности данных — клиенты видят чужие данные
3. API партнёров работают без политик безопасности
4. ИБ-специалист изолирован от процесса принятия решений

**Рекомендации:**
1. Внедрить единую IdP (расширить Keycloak на всю компанию)
2. Разработать политику классификации и защиты данных
3. Реализовать ABAC для API партнёров с проверкой tenant_id
4. Настроить SIEM и интегрировать ИБ в процессы изменений
